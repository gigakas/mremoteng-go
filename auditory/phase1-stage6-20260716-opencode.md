## 1. Code quality

- One declarative descriptor table drives the published header, rows and parsing (`internal/serialize/csv/columns.go:31`), preventing independent value/inheritance order drift.
- Serialization accepts a root or subtree and uses C# postorder semantics (`internal/serialize/csv/csv.go:44`). Deserialization is header-driven, reconstructs parents and tolerates unknown/malformed values like the original (`csv.go:144`, `csv.go:289`).
- Schema validation checks every reflection target and the merged-header positional quirk before serialization (`csv.go:470`), converting model drift into `ErrInvalidSchema` instead of panic.
- Eight tests cover exact C# header bytes, defective-schema round trip, credential filters, inherited effective passwords, subtree output, tolerant input, corrected aliases, reflection validation and semicolon sanitization (`csv_test.go:12-197`).

## 2. Performance

- Serialization/deserialization are O(rows × published columns). Reflection is bounded by the fixed schema and occurs only during file I/O, not protocol/UI hot paths.
- Parent reconstruction uses ID maps for O(1) lookup; postorder rows can attach to detached containers before those containers reach the root.
- Scanner capacity is bounded at 16 MiB per line, preventing unbounded token growth while supporting large connection records.

## 3. Architecture

- Changes are confined to stage-owned `internal/serialize/csv/`; no dependencies or cross-package edits.
- The canonical writer deliberately reproduces the C# merged `RedirectDiskDrivesCustomRedirectPorts` heading, missing final header semicolon with inheritance, and shifted inheritance tail. The reader compensates so values/flags stay synchronized, satisfying the blueprint while remaining able to import C# output.
- Corrected split-column aliases are accepted for forward compatibility without changing canonical writer output.
- Primary credential columns follow the C# SaveFilter; secondary VNC/gateway credentials remain structurally and behaviorally compatible.
- Intentional deviation: semicolons are removed from every value, including passwords. C# fails to sanitize Password and can corrupt row alignment; reproducing that defect would violate the blueprint's synchronization requirement.
